# Contributing to the Clinical NLP Dataset Registry

Thank you for your interest in contributing to the Clinical NLP Dataset Registry, maintained by the AMIA NLP Working Group.

This registry provides a standardized, FAIR-aligned catalog of clinical and biomedical NLP datasets to support transparency, reproducibility, and dataset discoverability.

We welcome community contributions.

---

## What Can Be Contributed?

You may suggest:

- New clinical or biomedical NLP datasets
- Corrections to existing metadata entries
- Updates to access conditions or licensing information
- Clarifications or improvements to documentation

---

## How to Suggest a New Dataset

All dataset submissions should be made via GitHub Issues.

1. Go to the **Issues** tab.
2. Click **New Issue**.
3. Select the **Dataset Submission** template.
4. Complete all requested metadata fields.

Please provide:

- Dataset name
- Official URL or repository link
- Primary NLP task(s)
- Clinical setting(s)
- Language
- Access type (Open / Restricted)
- Year of release
- Brief description
- Relevant publication (if available)

Submissions are reviewed by maintainers before inclusion.

---

## Inclusion Criteria

To be included, a dataset should:

- Be relevant to clinical or biomedical NLP
- Have publicly verifiable documentation
- Clearly state access conditions
- Provide sufficient information to validate the task and scope

The AMIA NLP Working Group reserves the right to edit submissions for clarity and consistency.
